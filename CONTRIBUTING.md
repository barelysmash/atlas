# Contributing to Atlas

## Principles
- Evidence over inference.
- Events over snapshots where auditability matters.
- Configuration over customization.
- Metrics are defined once.
- Human judgment remains central.

## Branching
- main: stable
- develop: integration
- feature/*: new work
- hotfix/*: urgent fixes

## Commit Style
Use conventional commits:

feat(events): add event model
fix(metrics): correct YoY calculation
docs(adr): record event sourcing decision
refactor(repo): reorganize packages
