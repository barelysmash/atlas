# RestaurantOS Design System

## Version 1.0

RestaurantOS should feel calm, intentional, and trustworthy. Visual design serves decisions, never decoration.

## Core Rules

1. Every page answers one primary question.
2. Every report ends with a recommended action.
3. Every chart must influence a decision or be removed.
4. Executive dashboards fit on one screen.
5. Default to black, white, and gray with a single accent for alerts.
6. Use whitespace generously.
7. Prefer trends over snapshots.
8. Show evidence before recommendations.
9. Keep terminology consistent.
10. Make refreshing data easier than editing data.
